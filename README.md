# The RAPID-MFA project
RAPID-MFA: *R*eusable *A*lgorithms and *P*rocesses for *I*ntegrated *D*ynamic *M*aterial *F*low *A*nalysis

## What is RAPID-MFA?
RAPID-MFA is a portfolio of dynamic MFA model variants.
It aims to provide **easily understandable** examples that **just work** of common usages of dynamic MFA models.

Including:
-Python code snippets
-Simple excel-based models
-Tutorials

The RAPID-MFA portfolio has two objectives:
1. Make dynamic MFA accessible, streamlined, flexible and easily adaptable
2. Serve as teaching material for self-learning and for courses such as [the MFA2 course in the CML - Leiden University](https://studiegids.universiteitleiden.nl/courses/115961/material-flow-analysis-ii)

## Project principles

The portfolio emphasizes clarity, simplicity, and being self-explanatory by aiming to stick to the following principles:

1. Traceability: All Python codes share the same basic codebase, copied and modified from the basic flow-driven codes and stock driven model codes.
2. Clarity over efficiency: minimal use of "automation" such as for loops, defining and calling functions, etc. Users can always add those themselves.
3. Transparency: no use of custom objects/classes or other "black boxes"
4. Explicit over implicit: thoughtful use of common Python packages including Pandas, Numpy, Scipy, and Seaborn for straightforward and simple tasks, but not as shortcuts for fundamental dynamic MFA operations
5. Modularity: different code files should be easily swappable, combined, and daisy-chained by simple copying and pasting of code snippets.
